Replicating the BDBV line-list delay model ​

Introduction ​

This walkthrough replicates the delay model of the epiforecasts/bdbv-linelist-analysis study of the 2012 Isiro Bundibugyo ebolavirus (BDBV) outbreak, component-by-component, on the composable CensoredDistributions.jl stack.

The original study hand-rolls four doubly-interval-censored delay fits, a HCW/non-HCW stratification, a death-pathway length-of-stay mixture, two convolved natural-history marginals, and a separate case-fatality logistic regression. Here we express each delay component through the package distributions and the Turing submodel front-end, fit a synthetic line list through the same path, and recover the generating parameters.

The aim is to show that the delay half of the model drops onto the package with no bespoke likelihood code. A whole record is ONE composed distribution: each case's pathway is a Sequential chain of doubly-censored edges, and the model scores it through ONE primary_censored_model(chain, row) submodel, letting the chain manage the per-edge condition-or-marginalise dispatch internally. The case-fatality regression is a covariate GLM and stays plain Turing, alongside the delay block. A shared-origin joint latent-time tree was deliberately rejected by the original study (NUTS divergences at same-day admissions, see its LIMITATIONS.md), so we do not build one; we note where it would sit and why it is out of scope.

The doubly-censored delay edges ​

The BDBV model fits four delay edges, each doubly interval censored at the day level (a primary event uniform over a one-day window, followed by a one-day secondary observation interval). The edges are onset→admission, admission→death, admission→discharge, and onset→notification.

On the package, each edge is a single double_interval_censored distribution. The default primary_event = Uniform(0, 1) and interval = 1.0 reproduce the day-level censoring the study uses.

julia
# An edge is one doubly-interval-censored distribution. We use the Gamma family
# throughout (the study's WAIC winner), parametrised by mean and shape so the
# priors are comparable across edges.
build_delay(log_mean, log_shape) = Gamma(exp(log_shape), exp(log_mean) / exp(log_shape))

function delay_edge(log_mean, log_shape)
    double_interval_censored(build_delay(log_mean, log_shape); interval = 1.0)
end

A record is one composed distribution ​

A whole record is ONE composed distribution: the case's pathway through the events, built from the edges with the Sequential composer. A case that dies follows onset→admission→death, so its pathway is Sequential(edge_oa, edge_ad); a case that is discharged follows onset→admission→discharge, Sequential(edge_oa, edge_ac). The notification edge is a separate one-step chain onset→notification.

A Sequential chain scored against a row of event TIMES conditions on each observed edge's own (censored) gap and marginalises any unobserved intermediate (#329), so the whole pathway is fitted through ONE submodel that manages the per-edge likelihood internally — no per-edge dispatch in the user model. Putting onset→admission inside both the death and the discharge chains fits it from every admitted case, while admission→death and admission→discharge are each fitted only from the cases that take that branch, matching the study's per-edge sample sizes with no double counting.

Simulating a synthetic line list ​

A line list row is a NamedTuple keyed by event name, which is exactly the data interface the package Turing models expect. We record event TIMES (onset at day 0, then the admission, death-or-discharge, and notification days), drawing each edge gap from its doubly-censored distribution with rand so the simulated days carry the same primary-window and daily-interval censoring the fit assumes. Each case takes the death or the discharge branch, and we tag HCW status, the case-definition flag, a standardised age, and the binary outcome for the CFR block.

julia
# Generating ("true") delay parameters: (log_mean, log_shape) per edge.
# A HCW gets a multiplicative shift on the mean of each edge.
truth = (
    oa = (log(3.0), log(2.0)),
    ad = (log(6.0), log(2.5)),
    ac = (log(13.0), log(2.0)),
    on = (log(7.0), log(1.8))
)
β_hcw_true = (oa = -0.3, ad = -0.2, ac = -0.25, on = -0.15)

# CFR generating coefficients (intercept, HCW, probable-case, age).
cfr_true = (β0 = 0.8, βhcw = -0.6, βdef = 0.4, βage = 0.3)

function sim_linelist(rng, n)
    rows = NamedTuple[]
    for _ in 1:n
        hcw = rand(rng) < 0.25
        shift = hcw ? 1.0 : 0.0
        # Draw each day-resolution edge gap straight from the doubly-censored
        # distribution via `rand`, so the simulated data follow exactly the
        # generative process the fit assumes (primary-window jitter, then the
        # delay, then the daily observation interval), giving unbiased recovery.
        gap(c) = rand(rng, delay_edge(
            truth[c][1] + shift * β_hcw_true[c], truth[c][2]))
        probable = rand(rng) < 0.4
        age_z = randn(rng)
        η = cfr_true.β0 + cfr_true.βhcw * hcw +
            cfr_true.βdef * probable + cfr_true.βage * age_z
        outcome = rand(rng) < logistic(η)
        # Event times along the case's pathway. Onset is the origin (day 0); the
        # case either dies (death observed, discharge missing) or is discharged.
        onset = 0.0
        admit = onset + gap(:oa)
        death = outcome ? admit + gap(:ad) : missing
        discharge = outcome ? missing : admit + gap(:ac)
        notif = onset + gap(:on)
        push!(rows,
            (
                onset = onset, admit = admit, death = death,
                discharge = discharge, notif = notif,
                hcw = hcw, probable = probable, age_z = age_z,
                outcome = outcome))
    end
    return rows
end

rng = Random.MersenneTwister(20260519)
linelist = sim_linelist(rng, 300)
first(linelist, 3)

Count / multiplicity weighting ​

At day-level resolution many cases share the same event-time combination. The study compresses repeated records into unique combinations with integer multiplicities and weights one likelihood evaluation per unique combination. On the package the multiplicity rides in the row as a reserved weight (or count) field, which the front-end excludes from the event dispatch and applies to the chain likelihood.

We compress the records of one pathway into one weighted row per unique combination of its event times.

julia
# Compress records into one weighted NamedTuple row per unique event-time
# combination. Each row carries the named event fields the chain spans (in event
# order) plus the reserved `weight` count, and only records whose pathway has all
# those events are included.
function pathway_rows(records, events::NTuple{N, Symbol}) where {N}
    counts = Dict{NTuple{N, Float64}, Int}()
    for r in records
        any(e -> r[e] === missing, events) && continue
        key = ntuple(i -> Float64(r[events[i]]), N)
        counts[key] = get(counts, key, 0) + 1
    end
    return [merge(NamedTuple{events}(k), (; weight = c)) for (k, c) in counts]
end

# The onset→admission→death pathway spans events (onset, admit, death).
death_rows = pathway_rows(linelist, (:onset, :admit, :death))
length(linelist), length(death_rows)

Fitting one pathway through ONE submodel ​

A record's pathway is one Sequential chain, scored through ONE primary_censored_model(chain, row) submodel. The chain conditions on each observed edge's own censored gap internally, so the user model never touches per-edge likelihoods; weight scales the chain contribution, and prefix namespaces each unique combination.

The edge priors match the study: a Normal on the log-mean centred on a plausible median, and a Normal on the log-shape.

julia
@model function fit_death_pathway(rows)
    lm_oa ~ Normal(log(3.0), 1.0)
    ls_oa ~ Normal(0.0, 1.0)
    lm_ad ~ Normal(log(6.0), 1.0)
    ls_ad ~ Normal(0.0, 1.0)
    # The whole pathway is ONE composed distribution.
    chain = Sequential(delay_edge(lm_oa, ls_oa), delay_edge(lm_ad, ls_ad))
    for (i, r) in enumerate(rows)
        obs ~ to_submodel(
            prefix(primary_censored_model(chain, r; weight = r.weight),
                Symbol(:rec, i)), false)
    end
end

death_fit = sample(fit_death_pathway(death_rows), NUTS(0.8), 600;
    progress = false)
mean(death_fit[:lm_oa]), truth.oa[1]
(1.0503007689909152, 1.0986122886681098)

The onset→admission log-mean recovers its generating value (log(3) ≈ 1.10) from the death pathway, even though that single submodel also fitted the admission→death edge.

Replicating the full atomic-delay block ​

The full delay block fits all four edges in one model, with each record's pathway scored through ONE chain submodel. Death cases take the Sequential(edge_oa, edge_ad) chain on (onset, admit, death); discharge cases take Sequential(edge_oa, edge_ac) on (onset, admit, discharge); the notification chain Sequential(edge_on) on (onset, notif) covers every case. This is the delay half of bdbv_model: onset→admission is shared by both admission chains, so it is fitted from every admitted case, while admission→death and admission→discharge are each fitted only from their branch.

julia
# Build the per-pathway weighted rows once.
rows_death = pathway_rows(linelist, (:onset, :admit, :death))
rows_discharge = pathway_rows(linelist, (:onset, :admit, :discharge))
rows_notif = pathway_rows(linelist, (:onset, :notif))

@model function delay_block(rows_death, rows_discharge, rows_notif)
    lm_oa ~ Normal(log(3.0), 1.0)
    ls_oa ~ Normal(0.0, 1.0)
    lm_ad ~ Normal(log(6.0), 1.0)
    ls_ad ~ Normal(0.0, 1.0)
    lm_ac ~ Normal(log(13.0), 1.0)
    ls_ac ~ Normal(0.0, 1.0)
    lm_on ~ Normal(log(7.0), 1.0)
    ls_on ~ Normal(0.0, 1.0)

    edge_oa = delay_edge(lm_oa, ls_oa)
    edge_ad = delay_edge(lm_ad, ls_ad)
    edge_ac = delay_edge(lm_ac, ls_ac)
    edge_on = delay_edge(lm_on, ls_on)

    # Each record's pathway is ONE composed distribution scored through ONE
    # submodel; the chain manages the per-edge condition/marginalise internally.
    death_chain = Sequential(edge_oa, edge_ad)
    discharge_chain = Sequential(edge_oa, edge_ac)
    notif_chain = Sequential(edge_on)

    for (i, r) in enumerate(rows_death)
        obs ~ to_submodel(
            prefix(primary_censored_model(death_chain, r; weight = r.weight),
                Symbol(:death, i)), false)
    end
    for (i, r) in enumerate(rows_discharge)
        obs ~ to_submodel(
            prefix(
                primary_censored_model(discharge_chain, r; weight = r.weight),
                Symbol(:disch, i)), false)
    end
    for (i, r) in enumerate(rows_notif)
        obs ~ to_submodel(
            prefix(primary_censored_model(notif_chain, r; weight = r.weight),
                Symbol(:notif, i)), false)
    end
end

Strata: HCW versus non-HCW ​

The stratified model shares the shape parameter within an edge but lets HCW and non-HCW cases differ on the log-mean, via an additive HCW shift. On the package this builds a per-stratum chain (the baseline log-mean, optionally shifted by the HCW β) and scores each stratum's records through the same single primary_censored_model(chain, row) submodel; the only package objects are the composed chains built per stratum.

julia
# Stratify the onset→admission edge by HCW status. The edge is a one-step chain
# on the events (onset, admit); we compress each stratum to unique combinations.
oa_hcw_rows = pathway_rows([r for r in linelist if r.hcw], (:onset, :admit))
oa_non_rows = pathway_rows([r for r in linelist if !r.hcw], (:onset, :admit))

@model function stratified_oa(rows_hcw, rows_non)
    log_mean ~ Normal(log(3.0), 1.0)
    log_shape ~ Normal(0.0, 1.0)
    β_hcw ~ Normal(0.0, 0.5)
    # Per-group chains: the HCW group's mean is shifted by β_hcw.
    chain_hcw = Sequential(delay_edge(log_mean + β_hcw, log_shape))
    chain_non = Sequential(delay_edge(log_mean, log_shape))
    for (i, r) in enumerate(rows_hcw)
        obs ~ to_submodel(
            prefix(primary_censored_model(chain_hcw, r; weight = r.weight),
                Symbol(:hcw, i)), false)
    end
    for (i, r) in enumerate(rows_non)
        obs ~ to_submodel(
            prefix(primary_censored_model(chain_non, r; weight = r.weight),
                Symbol(:non, i)), false)
    end
end

strat_fit = sample(stratified_oa(oa_hcw_rows, oa_non_rows), NUTS(0.8), 600;
    progress = false)
mean(strat_fit[:β_hcw]), β_hcw_true.oa
(-0.40514414228374557, -0.3)

The HCW shift on the onset→admission mean recovers its generating value.

The death-pathway length-of-stay mixture via Competing ​

The study's in-hospital length of stay is a mixture of the admission→death and admission→discharge delays, weighted by the in-hospital fatality among admitted cases. Competing expresses exactly this disjunction: exactly one outcome occurs (the case dies or is discharged), governed by a branch probability, and the node lowers to a MixtureModel over the two delays whose realisation is the single observed length of stay.

julia
# In-hospital fatality among admitted cases (the mixing weight).
p_die = count(r -> r.outcome, linelist) / length(linelist)

los = Competing(
    :death => (build_delay(truth.ad...), p_die),
    :disch => (build_delay(truth.ac...), 1 - p_die))

# The length-of-stay marginal: a single time-to-departure distribution.
mean(los), mean(as_mixture(los))
(8.239999999999998, 8.239999999999998)

Because a Competing is univariate it nests inside the other composers and scores a single observed time, so it drops into a Turing model through the same front-end as a leaf delay.

Convolved natural-history marginals via convolve_distributions ​

The natural-history identity onset→death = onset→admission ⊕ admission→death is recovered as a convolution of the fitted atomic components. convolve_distributions builds the distribution of the sum of independent delays directly, replacing the study's per-draw Monte-Carlo convolution in post-processing.

julia
onset_to_death = convolve_distributions(
    build_delay(truth.oa...), build_delay(truth.ad...))
onset_to_discharge = convolve_distributions(
    build_delay(truth.oa...), build_delay(truth.ac...))

# The convolution is a first-class distribution: `rand` draws the summed delay,
# so its marginal mean is summarised by sampling (as the study does for the
# Gamma convolutions, which have no closed form).
conv_mean(d; n = 50_000) = mean(rand(rng, d, n))
conv_mean(onset_to_death), conv_mean(onset_to_discharge)
(8.983125541087396, 16.049310662622865)

The convolved means equal the sums of the component means, as expected. The same Convolved object can itself be censored (double_interval_censored(convolve_distributions(d1, d2))) or used directly in a likelihood, so the marginal is a first-class distribution rather than a post-processing summary.

The CFR block: plain Turing, out of grammar ​

The case-fatality ratio is a covariate logistic regression on the HCW flag, the case-definition flag, and standardised age. This is a covariate GLM, not a censored delay, so it stays plain Turing and sits alongside the delay block rather than inside the distribution grammar. We state this explicitly: covariate regression is deliberately out of the CensoredDistributions grammar.

julia
# The binary outcomes are passed as a model variable (a vector observation), the
# standard Turing GLM idiom, with the covariates carried alongside.
@model function cfr_block(outcome, hcw, probable, age_z)
    β0 ~ Normal(0.0, 2.0)
    βhcw ~ Normal(0.0, 1.0)
    βdef ~ Normal(0.0, 1.0)
    βage ~ Normal(0.0, 1.0)
    for i in eachindex(outcome)
        η = β0 + βhcw * hcw[i] + βdef * probable[i] + βage * age_z[i]
        outcome[i] ~ Bernoulli(logistic(η))
    end
end

cfr_fit = sample(
    cfr_block(
        [r.outcome for r in linelist], [r.hcw for r in linelist],
        [r.probable for r in linelist], [r.age_z for r in linelist]),
    NUTS(0.8), 600; progress = false)
mean(cfr_fit[:βhcw]), cfr_true.βhcw
(-0.2899990757867953, -0.6)

The rejected alternative is a shared-origin joint tree: a single latent onset time per case feeding the admission and notification observations, with ordering constraints. The original study prototyped this and abandoned it (308 divergent transitions at same-day admissions, see its LIMITATIONS.md). The package can express it (a latent-wrapped Parallel couples branches through one shared origin), but we deliberately do not build it here: the marginal formulation matches what the study ships and what Charniga et al. (2024) recommend for retrospective complete-outbreak data.

Simulate and recover ​

We close with the headline check: fit the full atomic-delay block to the synthetic line list and confirm the credible intervals cover the generating log-means. The atomic edges have tight posteriors at this sample size, so we check the 99% credible interval, which keeps the single-seed recovery robust to Monte Carlo noise while still being a meaningful coverage statement.

julia
block_fit = sample(
    delay_block(rows_death, rows_discharge, rows_notif),
    NUTS(0.8), 1500; progress = false)

# 99% credible interval for each fitted log-mean and the generating value.
function covers(chain, sym, truth_val)
    draws = vec(chain[sym])
    lo, hi = quantile(draws, 0.005), quantile(draws, 0.995)
    return lo ≤ truth_val ≤ hi, (lo, hi)
end

recovery = (
    oa = covers(block_fit, :lm_oa, truth.oa[1]),
    ad = covers(block_fit, :lm_ad, truth.ad[1]),
    ac = covers(block_fit, :lm_ac, truth.ac[1]),
    on = covers(block_fit, :lm_on, truth.on[1]))

# Assert every credible interval covers its generating value.
@assert all(r -> r[1], values(recovery))
recovery
(oa = (true, (0.940127344986835, 1.1648382102668249)), ad = (true, (1.6876352160924488, 1.9242891129593804)), ac = (true, (2.3558097721410745, 2.717375622434418)), on = (true, (1.7213582539759984, 1.9503916302004642)))

Every delay log-mean credible interval covers its generating value, so the package replication recovers the truth from a simulated line list.

Summary ​

The delay half of the BDBV model maps cleanly onto the rebuilt stack:

  • each delay edge is a double_interval_censored distribution;

  • a record's pathway is ONE Sequential chain of those edges;

  • the whole pathway is fitted through ONE primary_censored_model(chain, row) submodel that manages the per-edge condition/marginalise internally — no per-edge dispatch in the user model;

  • strata build a per-group chain and score it through the same submodel;

  • the death-pathway length of stay is a Competing node;

  • the natural-history marginals are convolve_distributions;

  • count weighting rides in the reserved weight field of each row.

The case-fatality regression stays plain Turing as an explicit out-of-grammar covariate GLM, and the shared-origin joint tree is left unbuilt by design.
